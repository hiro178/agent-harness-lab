# A4: voicevox-agent-notify — VOICEVOX 音声通知プラグイン

- 作成日: 2026-07-12 / 工数見積: 1 人日 / リスク: 低（国内特化）
- 種別: ルール + MCP 設定ガイドのプラグイン
- 公開言語: **日本語優先**（README は日本語、英語サマリを末尾に付ける）

## 目的

作業の節目で Claude Code がキャラクターボイスで状況報告する体験を丸ごとパッケージ化する。日本語圏ニッチの独占枠。「エージェントが喋る」体験はデモ動画映えするため、X での動画付きドロップと相性が良い。

## 抽出元資産

- `~/.claude/rules/voicevox/voice-notification.md` — 報告タイミング表（7 フェーズ）、口調ガイドライン、発話多様性ルール
- `~/.claude/settings.json` の VOICEVOX MCP 配線部分（該当ブロックのみ抜粋）

## 公開物

1. `plugins/voicevox-agent-notify/rules/voice-notification.md` — キャラクター非依存に汎用化した版（話者 ID・口調をユーザー設定に切り出す）
2. `plugins/voicevox-agent-notify/README.md` — VOICEVOX 本体のセットアップ、MCP サーバー登録手順、話者カスタマイズ方法
3. デモ GIF/動画（発話タイミングが分かる 30 秒程度）

## 実装ステップ

1. ルールから特定キャラクター固有部分（口調例文）を「例: もち子さん設定」としてサンプル節に隔離し、素の構造（フェーズ × タイミング表）を本体とする
2. 使用する VOICEVOX MCP サーバー実装を特定し、README にインストール手順を書く（自作なら同梱、サードパーティなら依存として明記）
3. VOICEVOX と各キャラクターの利用規約を確認し、クレジット表記義務（例:「VOICEVOX:○○」）とキャラクター音声の利用条件を README に明記する
4. クリーン環境（ルール未導入のセッション）で導入手順どおりにセットアップし、発話が機能するか実走検証
5. デモ動画を撮って README に埋め込む
6. plugin.json + marketplace.json 登録

## 注意点

- **ライセンス確認が最重要**: VOICEVOX 本体は OSS だがキャラクター音声の利用規約は個別。再配布するのは「設定とルール」だけで音声モデルは含めない構成とし、規約リンクを明記する
- 英語圏ユーザー向けには「Works with any TTS MCP server」の一般化余地を README 末尾に示す（将来ユニット化の布石）
