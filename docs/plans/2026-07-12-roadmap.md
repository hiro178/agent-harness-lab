## 目的

AI エージェントによる設計・実装を支援する小粒ユニット（スキル / フック / エージェント仕様 / CLI）を、単一のプラグインマーケットプレイスリポジトリに順次公開し、GitHub スターを複利で獲得する。

## 戦略決定の根拠

市場調査（2026-07-12 実施）で得たシグナル:

1. スキル大量集約リポジトリ（345 スキル集等）は飽和しており「数」では差別化できない
2. 単一ファイル・単一機能でも観察に基づく知見があれば爆発する（4 行動原則のみの CLAUDE.md が 15 万★）
3. superpowers（94K★）・Everything Claude Code（100K★）はいずれも単一リポジトリ — スターは 1 リポジトリに複利で積み上がる
4. 名前空間調査: `harness-engineering` 系は激戦区（770 件・トップ 10K★）、`agent-harness-lab` はほぼ空き地（最大★4）

→ 結論: **単一リポジトリ + ユニット順次公開**。各ユニットの公開が「コンテンツドロップ」となり SNS 再拡散の起点になる。マルチリポジトリ案（9 分散）は管理コスト 9 倍・スター分散のため棄却。

## 公開運用ルール

- リポジトリは**A1 完成まで private**。A1 完成時に public 化し、以後ユニット完成ごとに GitHub Release + SNS 投稿（X / Reddit r/ClaudeAI）
- 公開言語は英語（README・SKILL.md）。A4 voicevox のみ日本語優先（国内ニッチ独占狙い）
- ライセンス: MIT
- awesome-claude-code / awesome-claude-skills への掲載 PR を A3 公開時点（3 ユニット揃い）で提出
- 出所規律: 抽出元の知見に外部ソース由来のもの（LayerX・Anthropic 公式・aidlc-workflows 等）が含まれる場合、必ず出典を README に明記し、独自成果と輸入知見を区別する
