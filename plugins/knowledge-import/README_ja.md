# knowledge-import plugin

*[English](README.md) | [日本語](README_ja.md)*

外部のコンテンツをプロジェクトに取り込む際の安全チェック —— 「リンクを読んで要約する」だけではない。

## 素朴なやり方の何が問題か

エージェントに「この記事をルールとして取り込んで」とお願いすると、喜んでブログ記事を1本読んで、それだけを根拠にプロジェクトの指針を書き換えてしまう。裏取りされていない情報源1つに、プロジェクト全体の動き方を決める書き込み権限を渡してしまうということだ。このプラグインは、欠けていた安全チェックを追加する: **コンテンツが実際の変更になるのは、いくつかのチェックを通過してからだけ** —— この情報源は本当に信頼できるか、独立した2つ目の情報源も同じことを言っているか、似たものが既に存在していないか —— これらすべてが自動で強制される。プロンプトに書かれただけの、理屈をつけて回避できる「提案」ではない。

## 「リンクを読んで要約する」との違い

- **情報源の信頼度をチェックし、裏付けを求める。** 情報源にはランクが付く（公式ドキュメントや査読済み論文が最上位、素性の知れない個人ブログが最下位）。エージェントが自分の都合で情報源のランクをこっそり上げてしまわないようルールで縛ってある。ランクの低い情報源1つだけでは何も変更できず、独立した2つ目の情報源が同じことを言っているか、あなたが明示的に「それ1つだけで進めていい」と承認するかのどちらかが必要になる。
- **通常は混同されがちな3つの結果をきちんと区別する。** 既に持っているものを新しい知見で拡張する、まったく新しいものを作る、あるいは（取り込む内容がアイデアではなくチェックリストや基準の場合）**実際のプロジェクトをそれに照らしてチェックし、合っていない部分を直す**（すべての変更を事前にプレビューし、バックアップを取り、リスクのある変更には承認を必須にする）。
- **完全に重複するものは、はっきりブロックする。** 気づかないうちに二重に作られることがない。
- **最終チェックを通らないと何も反映されない。** 書式・構造・「もう存在していないか」・プロジェクトの既存ルールに沿っているか、をすべて確認する。

## インストール

```text
/plugin marketplace add hiro178/agent-harness-lab
/plugin install knowledge-import@agent-harness-lab
```

## 発動タイミング

外部コンテンツをプロジェクトに取り込みたい・統合したいと頼んだとき、あるいはどこかで見つけたチェックリストや基準に照らして、実際の何か（設定ファイルやコード）を確認したいときに発動する。

具体的な手順（情報源のランク付け方法・コンテンツの分類方法・変更の安全な適用方法）は [`skills/knowledge-import/SKILL.md`](skills/knowledge-import/SKILL.md) とその `references/` フォルダにある。
