# knowledge-import plugin

*[English](README.md) | [日本語](README_ja.md)*

外部コンテンツをプロジェクトの成果物へと変換するパイプライン — 「要約して保存」ではなく、出所の三角測量をゲートとする。

## ナイーブな知見取り込みの問題点

エージェントに「この記事をルールとして取り込んで」と頼めば、喜んでブログ記事を1本読んで CLAUDE.md を書き換えてしまう。1つの未検証ソースがプロジェクトの設計判断への書き込み権限を持ってしまうということだ。本プラグインは欠けていたゲートを追加する: **コンテンツは出所の三角測量・重複排除・構造検証を通過して初めて生成に到達する** — すべて制御フローとして強制され、モデルが理屈をつけて回避できるプロンプト上の提案ではない。

## 「リンクを読んで要約する」との違い

- **権威階層 + 三角測量**（Phase 1）: ソースは明示的な決定表に基づき primary/secondary/tertiary に階層化され、モデルがソースの階層を自己昇格できないようアンチゲーミング制約が入る。単一ソースの tertiary な取り込みは、2つ目の独立ソースが見つかるか、ユーザーが明示的にダウングレードを承認するまで、何も変更できないようブロックされる。
- **統合 vs 新規成果物 vs 監査・是正**（Phase 3.5/3.6）: 通常は混同されがちな3つの結果をパイプラインが区別する — 既存ルールを新しい失敗モードで拡張する、独立した新規成果物を生成する、あるいは（取り込んだ内容が規範的基準の場合）**実際の設定・コードをそれに照らして監査し乖離を是正する**（ドライラン・バックアップ・リスク分類された承認ゲート付き）。
- **重複排除をブロッキングゲートとして**（Phase 4）: 完全一致の取り込みは静かに重複させるのではなく、明確にブロックする。
- **何かが出荷される前の検証ゲート**（Phase 6）: frontmatter・構造・重複再チェック・規約準拠 — すべて fail-closed。

## インストール

```text
/plugin marketplace add hiro178/agent-harness-lab
/plugin install knowledge-import@agent-harness-lab
```

## 発動タイミング

外部コンテンツのプロジェクトへの取り込み・統合を依頼したとき、または取り込んだベストプラクティスのチェックリストで実際の成果物を監査したいときに発動する。

パイプラインの詳細（ルーティング表・分類マトリクス・是正手順）は [`skills/knowledge-import/SKILL.md`](skills/knowledge-import/SKILL.md) とその `references/` にある。
